# MyVim — Терминальный текстовый редактор

Проект «MyVim» представляет собой VIM-подобный текстовый редактор, разработанный на языке программирования Python 3. Главной целью проекта является реализация приложения на базе архитектурного шаблона Model-View-Controller (MVC) с применением текстового пользовательского интерфейса (TUI).

## Архитектура и паттерны проектирования

Программа строго разделена на независимые компоненты, что гарантирует изоляцию логики обработки данных от отрисовки интерфейса:

- **Model (Модель):** Отвечает за хранение текста, файловые операции (сохранение/загрузка) и алгоритмы редактирования. Текст обрабатывается с помощью низкоуровневой обертки над C++ классом `MyString`, что обеспечивает скорость операций в памяти. Модель реализует паттерн «Наблюдатель» (Observable), уведомляя подписчиков о любых изменениях данных.
    
- **View (Представление):** Отвечает за визуализацию текста, статусной строки и курсора. Реализует паттерн «Окно просмотра» (ViewPort) для эффективного скроллинга: вычисляется срез данных (`offset_row`), и клиенту отображается только та часть текста, которая помещается в экран.
    
- **Controller (Контроллер):** Представляет собой конечный автомат, который опрашивает клавиатуру, переключает режимы работы и интерпретирует VIM-команды (например, `dd` для удаления строки).
    
- **Паттерн «Адаптер»:** Для создания TUI используется встроенный модуль `curses`. Логика отображения отделена от конкретной библиотеки с помощью адаптера `CursesAdapter`, что позволяет при необходимости сменить графическую библиотеку, не меняя код Model и View.
    

## Режимы работы

Редактор поддерживает машину состояний с классическими для VIM режимами:

- **Normal (Режим навигации):** Основной режим, активный при старте программы. Клавиши интерпретируются как команды для перемещения курсора, копирования или удаления текста.
    
- **Insert (Режим ввода):** Предназначен для ввода текста в текущую позицию курсора. Активируется командами `i`, `I` (с новой строки) или `S` (с очисткой текущей строки). Выход осуществляется по клавише `<ESC>`.
    
- **Command-line (Режим команд):** Активируется клавишей `<:>`. Используется для работы с файловой системой (сохранение файла `:w`, выход `:q` или `:q!`).
    
- **Search (Режим поиска):** Позволяет искать подстроки в файле. Команда `</>` инициирует поиск до конца документа, а `<?>` — до его начала.
    

## Основной функционал

- **Информативная строка состояния:** Отображает текущий режим работы, имя открытого файла, номер текущей строки и общее количество строк в документе.
    
- **Поддержка кодировок:** Программа работает с 1-байтовой кодировкой текста.
    
- **Автоскроллинг:** Динамический расчет высоты окна и смещения позволяет корректно отображать файлы, содержащие сотни строк.
    

## Тестирование

Проект покрыт модульными тестами с использованием фреймворка для обеспечения надежности базового функционала.

- Тесты проверяют инициализацию состояния, парсинг команд (например, сохранение `:w\n`), логику буфера обмена (`yy` и `p`) и переключение режимов конечного автомата.
    
- Применение Mock-объектов позволило протестировать логику Model и Controller без необходимости запуска реального окна терминала
